## [1.5.2]
### New
- USB プロトコル経由の UPS デバイスのサポートを追加
- ファイル内の HEIC 画像の Live Photo プレビューと自動再生を追加しました
- ワンクリックアプリケーションキャッシュクリーンアップ機能を追加
- 組み込みの Markdown エディターを追加しました

### Fixed
- USBドライブをシステムディスクとして使用する場合、ファイルに「ZimaOS-HD」が表示される異常を修正
- USB ドライブから起動するときにバックアップで zima-storage が開かない問題を修正しました。
- ディスクタイプの解析エラーによりストレージデバイスがファイルに表示されない問題を修正しました
- マウントポイントのパスにスペースが含まれている場合の表示異常を修正
- スペースを含むディレクトリを移行するときの異常なサイズ計算を修正しました
- docker-cli を使用してプロジェクトをインポートする際のプロジェクト名の競合を修正しました
- avahi-daemon サービスの予期しない終了を引き起こす安定性の問題を修正しました
- ストレージ作成時に選択可能なデバイスとして mmcblkboot パーティションが誤ってリストされる重大なバグを修正しました (これらはシステムで予約されたブート パーティションであり、決してフォーマットしてはなりません)
- ファイルのカット/ペーストの競合解決中にファイルで不正確な「スキップ」動作が修正されました。
- Samba で欠落している CP850 コードページのサポートを修正しました (Windows クライアントでファイル名が文字化けする原因となっていました)

### Optimized
- Improved default prompt messages after importing containers via docker run
- 最適化された LocalStorage インデックス ロジック
- 初期バックアップモジュールインターフェースとタスク管理インターフェースを最適化しました。
- バックアップ ポリシーの説明がより正確かつ明確になるように改良されました。
- ネットワーク ウィジェットのスケールが動的に調整されるようになりました
- 最適化されたアプリストアの推奨ロジック
- 選択可能なバックアップ元範囲の拡大
- バックアップ戦略の文言が洗練されて読みやすくなりました
