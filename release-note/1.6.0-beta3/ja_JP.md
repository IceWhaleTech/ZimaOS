## [1.6.0-beta3]

### Fixed
- デバイスIDの取得方法を修正
- カスタマイズされたディスクが完全に表示されない問題を修正しました
- z-index 階層化に関するいくつかの問題を修正しました
- vfat タイプのディスクの存在によりストレージが不足しているように見える問題を修正しました
- 有効なハードドライブをフォーマットするとデータベースに正確に保存されない問題を修正しました
- ごみ箱の不正確な表示を修正しました

### Improved
- 欠落している RAID メンバー ディスクを判断するためのロジックを最適化しました。
- カスタマイズされたディスクの保存ロジックを最適化しました。



## [1.6.0-beta2]

### Fixed
- 正確なディスク インデックスが確率的に取得できない問題を修正し、WebUI でディスクが時々消える問題を解決しました。
- パーティションのある USB デバイスが自動的にマウントできない問題を修正しました。
- 背景が透明なカスタムデバイス画像が黒く表示される問題を修正しました。

### Improved
- USB デバイス有効化ロジックを最適化しました。有効化する際のマウント ポイントの選択をサポートするようになり、柔軟性が向上しました。
- ストレージ有効化ボタンの表示ロジックを最適化して、見つけやすくしました。




## [1.6.0-beta1]

### New
- ストレージデバイスのカスタマイズ表示を追加しました
- USBクラスのストレージデバイス管理を追加しました
- RAIDメタデータの信頼性の高い保存機構を追加し、システム再インストールやデバイス交換後に元のアレイを自動で再認識してマウントできるようにしました
- デバイス画像とログイン壁紙のカスタマイズ表示を追加しました

### Fixed
- mod_management モジュールの pull 制限問題を修正しました
- SMB がクライアントとして動作する際の接続制限問題を修正しました
- SMB が想定どおりに「共有権限のないフォルダをスキップしない」問題を修正しました
- smartd サービスによる断続的なウェイクアップでディスクが通常のスリープに入れない問題を修正しました
- 異常な docker0 ネットワークインターフェースが原因で Docker の起動に失敗する問題を修正しました
- データテーブル欠落またはテーブル構造異常により Files サービスの起動に失敗する問題を修正しました

### Optimized
- ネットワークカード情報の表示を最適化しました
- 設定ページの表示を最適化しました
- ZimaOS の未ログイン時のリクエスト一覧処理を最適化しました
- アプリケーションの pull 失敗時のエラープロンプト文言を最適化しました
- 公式 ZimaCube デバイスで7番目のドライブの表示順を調整しました
